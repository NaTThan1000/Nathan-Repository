# Part 1: 游戏策划文档

## 1. 游戏概述

### 1.1 游戏定位
**类型**: 塔罗主题双人对战桌游 | **平台**: 网页 | **视角**: 人 vs AI

### 1.2 核心概念
双方轮流从牌堆抽牌，将小阿尔卡纳牌打入 4 个元素阵中比拼点数。抽到宫廷牌时额外抽一张大阿尔卡纳牌，大牌可产生强力效果影响场上局势。小牌堆耗尽时终局结算，4 阵逐阵记分，总分高者获胜。

### 1.3 设计目标
- 用塔罗牌的大小牌分离机制创造"稳定出牌 + 突发事件"的双层策略
- 大牌正/逆位提供 44 种不同效果，高重玩性
- 持续性效果叠加交错产生组合策略深度
- 单局约 15-25 分钟（56 张小牌轮流打出）

## 2. 系统设计

### 2.1 牌组系统

#### 小阿尔卡纳（56 张）— 稳定计分资源

| 花色 | 图标 | 数字牌(点数 1~10) | 宫廷牌(固定 10 点) |
|------|------|-------------------|---------------------|
| 权杖 | 🪄 | 权杖1 ~ 权杖10 | 侍从 / 骑士 / 皇后 / 国王 |
| 圣杯 | 🏆 | 圣杯1 ~ 圣杯10 | 侍从 / 骑士 / 皇后 / 国王 |
| 宝剑 | ⚔️ | 宝剑1 ~ 宝剑10 | 侍从 / 骑士 / 皇后 / 国王 |
| 星币 | 🪙 | 星币1 ~ 星币10 | 侍从 / 骑士 / 皇后 / 国王 |

**打出规则**:
- 数字牌 **必须打入相同花色** 的元素阵（教皇正位可例外）
- 宫廷牌可打入 **任意** 元素阵，且立即抽一张大牌预览
- 打出后点数加入该阵己方总分

#### 大阿尔卡纳（22 张）— 强力效果

每张大牌具有 **正位（upright）和逆位（reversed）** 两种效果，共 44 种效果。

**效果分类**:

| 类型 | 行为 | 代表牌 |
|------|------|--------|
| 一次性 (instant) | 执行效果后弃入弃牌堆 | 愚人、女皇、力量、命运之轮、正义、审判 |
| 持续性 (persistent) | 留在阵中持续发挥影响 | 魔术师、隐士、死神、星星、恶魔、世界 |
| 触发性 (trigger) | 条件满足时自动触发 | 恶魔正位、塔正位、女祭司逆位 |

**大牌效果速查**:

| ID | 名称 | 正位 | 逆位 |
|----|------|------|------|
| 0 | 愚人 | 选阵双方小牌全弃 | 选阵双方大牌丢弃 |
| 1 | 魔术师 | 己方小牌移至另阵 | 后续小牌打入其他阵(持续) |

| 6 | 恋人 | 抽2张选1保留给己方(人类弹窗/AI高分) | 抽 1 替换双方各 1 小牌 |
| 7 | 战车 | 敌方大牌失效(持续) | 复制敌方大牌效果(持续) |
| 8 | 力量 | 抽 2 小牌打出 | 阵中≥3→全弃+重抽到宫廷 |
| 9 | 隐士 | 弃牌堆顶 2+选弃/留 | 双方后续小牌打入对方阵(持续) |
| 10 | 命运之轮 | 己方小牌右移一格 | 己方小牌左移一格 |
| 11 | 正义 | 高分方最小牌移对方 | 牌多方最大牌移对方 |
| 12 | 倒吊人 | 己方大牌弃+抽大牌 | 敌方大牌弃+抽大牌 |
| 13 | 死神 | 阵中小牌全弃+重抽 | 双方非宫廷小牌丢弃(持续) |
| 14 | 节制 | 双方小牌洗混重分配 | 敌方出小牌弃阵中 1 牌(持续) |
| 15 | 恶魔 | 敌方小牌打入己方阵(触发) | 本阵全弃+弃大牌(持续) |
| 16 | 塔 | 排名变化→双方小牌弃(触发) | 各自最高分阵小牌丢弃 |
| 17 | 星星 | 出小牌时额外抽 1 张(持续) | 小牌打入敌方阵(持续) |
| 18 | 月亮 | 抽 2 背朝上终局翻开 | 双方阵小牌互换(持续) |
| 19 | 太阳 | 己方选任意 2 阵互换小牌(交互) | 按此阵胜负判定(持续) |
| 20 | 审判 | 全部阵所有大牌丢弃 | 己方大牌弃+抽 2 小牌 |
| 21 | 世界 | 1-10 全→判己方胜(持续) | 终局互换双方小牌(持续) |

### 2.2 元素阵系统

4 个元素阵分别对应 4 个花色，双方各占一边。每阵独立记分。

**阵属性**:
- `cards[]` — 该阵己方打出的小牌
- `score` — 当前总分（小牌点数之和）
- `activeMajor` — 当前生效的持续性/触发性大牌（每阵最多 1 张）
- `lockedScore` — 皇帝正位锁定分数（新牌仍累加）
- `majorHistory[]` — 该阵打出过的大牌历史记录

**数字牌限定**：数字牌只能打入**对应花色**的元素阵（魔术师逆位可强制改阵，教皇正位可多选）。

### 2.3 回合系统

```
当前玩家回合:
  1. startTurn() → 从小牌堆抽 1 张
  2. 女皇逆位检查 → 是 → 丢弃 → 回合结束
  3. 数字牌 → 玩家/ AI 选择元素阵打出 → endTurn()
  4. 宫廷牌 → 选阵打出 + 抽大牌预览 → 玩家决定:
     - 打出大牌（选正/逆位+目标阵） → endTurn()
     - 弃掉大牌 → endTurn()
  5. 回合结束 → 切换对手 → 循环
```

**终局触发**：小牌堆 `minorDeck.length === 0` 时立即进入 `endGame()`。

## 3. 数值设计

### 3.1 记分规则

| 环节 | 规则 |
|------|------|
| 小牌点数 | 数字牌 = 面值(1-10)，宫廷牌 = 固定 10 点 |
| 阵总分 | 阵中所有小牌点数之和 |
| 终局逐阵 PK | 胜者得 1 分，平局无人得分 |
| 皇帝逆位(4R) | 该阵阵 PK 胜者得 2 分 |
| 世界正位(21U) | 该阵 1-10 齐全 → 直接判己方胜该阵 |
| 总分平局 | 依次比较阵总点数 → 先手方 A 胜 |

### 3.2 AI 策略参数

| 决策 | 规则 |
|------|------|
| 大牌打出率 | 85% 概率打出，15% 概率弃牌 |
| 正逆位选择 | 70% 正位，30% 逆位 |
| 目标阵选择 | 选己方分数领先最多的阵；全落后则随机 |

1. 启动 → A 自动抽第一张小牌
2. 玩家仅可点击**己方(A)**元素阵（下排蓝色高亮区域）

6. 终局 → 弹窗展示 4 阵对决结果 → 宣布胜者

# Part 2: 技术架构速查

## 6. 架构概览

- 单一 `TarotGame` class 管理全部状态
- `MAJOR_DEFS` 内联数组定义 22 张牌效果（数据驱动，非 44 个方法）
- 效果分发：一次性 → `_execEffect()`，持续性/触发性 → `_setupActiveMajor()` + `_chkPersistent()`
- 回合驱动：`continueFlow()` → setTimeout 链（异步，非帧循环）
- UI 渲染：`render()` 全量 innerHTML 重绘
- 调试模式：`debugMode` / `debugSelectedCard` / `debugUsedCardIds`(Set) / `debugDrawnFromDeck` 全局标志，与正常回合隔离，提供随机抽牌、一键出牌、选抽牌 + 防重复 + 牌堆回放等完整测试能力

## 7. TarotGame 核心属性

```js
minorDeck / majorDeck          // 牌堆（已洗牌）
minorDiscard / majorDiscard    // 弃牌堆
currentPlayer                  // 'A' | 'B'
phase                          // 'play_minor' | 'play_major' | 'sun_swap' | 'lover_choice' | 'game_over'
drawnMinor / drawnMajor        // 当前待处理的牌
majorOrientation               // 'upright' | 'reversed'
players.A/B.elements[suit]     // { cards[], activeMajor, majorHistory[], score, lockedScore }
// 特殊状态标记
moonHiddenCards[]              // 月亮正位隐藏牌 [{player, element, cards}]
_empressRevFlag{A,B}           // 女皇逆位下次抽牌丢弃
_rankSnapshot                  // 塔正位排名快照
_chariotRevPending             // 战车逆位复制标记
_pendingDrawnMajor             // 倒吊人正位(12U)：当前玩家待选大牌
_deferredMajor                 // 倒吊人逆位(12R)：敌方待选大牌（延迟至敌方回合）
_pendingLoverCards             // 恋人正位(6U)：{player, element, cards[]} 待玩家/AI选择保留牌
_pendingSunSwap                // 太阳正位(19U)：{player, element1} 待选第二个阵
```

## 10. 数据流

```
startTurn()
  → 小牌堆 pop() → 女皇逆位丢弃检查
  → 宫廷牌? → 大牌堆 pop() 预览
  → AI/玩家选择

playMinorToElement(p, card, element)
  → 12 步持续性检查链（依次调用 _chkPersistent）
  → 更新 element.cards[] 和 score
  → 触发 _checkTowerTrigger()（排名快照对比）

majorDecision(play, element, orientation)
  → instant → _execEffect() + 弃牌
  → 检测 _pendingLoverCards (6U) / _pendingSunSwap (19U)
  → 人类等待选择 / AI自动处理
  → persistent/trigger → _setupActiveMajor() 留在阵中

endTurn() → 切换玩家 → 小牌堆空? → endGame → finalScore
```
