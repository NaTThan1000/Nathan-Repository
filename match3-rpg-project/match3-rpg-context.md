# Part 1: 游戏策划文档

## 1. 游戏概述

### 1.1 游戏定位
**类型**: 三消 + RPG 回合制战斗 | **平台**: 网页（手机端优先） | **视角**: 单人 PvE

### 1.2 核心概念
玩家在 8×8 棋盘上通过交换相邻宝石形成 3+ 连线消除，消除的宝石颜色会为对应英雄充能，充满后点击英雄释放技能攻击敌人。击败连续 4 波敌人即可通关。

### 1.3 设计目标
- 消除 → 充能 → 技能的反馈循环，让消除有战术目的
- 5 色 5 英雄对应，每种颜色消除都有意义
- 短局制（4波），适合手机碎片时间

## 2. 系统设计

### 2.1 棋盘系统

| 参数 | 值 |
|------|-----|
| 棋盘尺寸 | 8行 × 8列 |
| 宝石颜色 | 5种（火/冰/木/雷/光） |
| 消除条件 | 横或竖 3+ 同色连线 |
| 特殊生成 | 4连→火箭(清整行)，5连→炸弹(3×3范围) |
| 初始棋盘 | 无预生成匹配，保证开局公平 |

**操作方式**：支持鼠标拖拽和触摸滑动交换相邻宝石。无效交换自动回弹。

### 2.2 英雄系统

每位英雄对应一种宝石颜色，消除该色宝石为对应英雄充能。

| 槽位 | 名称 | 颜色 | HP | ATK | 充能需求 | 技能名 | 技能效果 |
|------|------|------|-----|-----|----------|--------|----------|
| 1 | 🛡️ 圣光骑士 | 光(紫) | 105 | 12 | 28 | 圣盾术 | 全体 +25 护盾 |
| 2 | ⚔️ 炎之战士 | 火(红) | 85 | 18 | 24 | 烈焰斩 | 单体 200% ATK 伤害 |
| 3 | 🧙 冰霜法师 | 冰(蓝) | 55 | 22 | 20 | 暴风雪 | 全体 150% ATK 伤害 |
| 4 | 🌿 森林贤者 | 木(绿) | 65 | 10 | 20 | 治愈之光 | 全体恢复 30 HP |
| 5 | ⚡ 雷霆刺客 | 雷(黄) | 50 | 28 | 16 | 致命一击 | 单体 300% ATK 暴击 |

**充能规则**：每消除 1 颗宝石为该颜色英雄充能 1 点。特殊宝石（火箭/炸弹）范围消除的宝石同样计入充能。

**释放时机**：充能满后英雄卡片闪烁提示，玩家可在 idle 动画阶段随时点击释放，不消耗回合。

### 2.3 敌人系统（4波）

| 波次 | 名称 | HP | ATK | 技能池 |
|------|------|-----|-----|--------|
| 1 | 🐉 暗影魔龙 | 250 | 10 | 龙爪(单体1.0×)、龙息(AOE 0.5×)、咆哮(全体 atk↓) |
| 2 | 👹 冰霜巨人 | 350 | 15 | 重锤(单体1.5×)、冰霜新星(AOE 0.6×)、冰甲(def↑)、冰冻(单体stun) |
| 3 | 🦂 毒蝎女王 | 450 | 20 | 毒刺(单体1.2×+中毒)、毒雾(AOE 0.5×+群体中毒)、狂暴(atk↑) |
| 4 | 👾 雷霆之王 | 600 | 25 | 雷击(单体1.8×)、连锁闪电(AOE 0.8×)、雷盾(def↑)、麻痹(群体stun) |

**AI 行为**：每 200 帧（约 3.3 秒）攻击一次，从技能池中随机选择一个执行。被眩晕时跳过回合。

### 2.4 状态效果系统

| 效果 | 适用对象 | 机制 |
|------|----------|------|
| 眩晕（⚡） | 敌人 | 持续 N 帧，期间跳过攻击 |
| 中毒（☠） | 英雄/敌人 | 每 30 帧扣除固定伤害，持续 N 帧 |
| 攻击↑（⚔） | 敌人 | 临时提升敌人伤害倍率，持续 N 帧 |
| 防御↑（🛡） | 敌人 | 临时降低受到伤害，持续 N 帧 |
| 攻击↓（⬇） | 敌人 | 临时降低敌人伤害倍率，持续 N 帧 |
| 护盾 | 全体英雄 | 吸收伤害，剩余值显示在界面上 |

## 3. 数值设计

### 3.1 伤害公式

| 伤害来源 | 公式 |
|----------|------|
| 基础消除伤害 | `存活英雄 ATK 总和 × 0.25 × combo 倍率` |
| combo 倍率 | `1 + (combo数 - 1) × 0.2` |
| 火箭伤害 | `对应英雄 ATK × 1.5 × combo 倍率` |
| 炸弹伤害 | `对应英雄 ATK × 2.5 × combo 倍率` |

### 3.2 得分公式

| 来源 | 分值 |
|------|------|
| 每次消除 | `消除宝石数 × 10 × combo 倍率` |
| 波次通关奖励 | `100 × 当前波次` |

### 3.3 英雄生存曲线

- 坦克型（圣光骑士 105HP/盾技能）：承受 2+ 轮集火
- 输出型（雷霆刺客 50HP/高输出/低充能）：高爆发高风陷
- 平衡型（炎之战士 85HP/炎之战士 18ATK）：稳定输出
- 续航型（森林贤者 65HP/治疗）：延长队伍存活
- 控制型（冰霜法师 55HP/AOE）：清场

## 4. 界面与交互

### 4.1 布局结构（手机 420px 宽）

```
┌──────────────────────┐
│ 状态栏装饰（刘海）      │
│           [⏸ 暂停]    │
│  🐉 敌方面板           │
│  ├ 头像 + 名称 + 波次  │
│  ├ HP 血条             │
│  └ 状态标签            │
│ ┌─ 8×8 棋盘 ────────┐ │
│ │ 48px/cell 八边形宝石 │ │
│ └────────────────────┘ │
│  得分:0    回合:1      │
│      COMBO!            │
│     消息区             │
│ [英雄1][英雄2]...[英雄5]│
│  HP条 / 充能条 / 技能  │
└──────────────────────┘
```

### 4.2 交互流程

1. **开始界面** → 点击"开始战斗"
2. **游戏中** → 拖拽/点击交换宝石 → 自动消除 → 充能满了点英雄放技能
3. **波次通关** → 弹窗显示成绩 → 点击"继续下一关"
4. **游戏结束** → 弹窗显示总分/存活数 → 点击"重新挑战"

### 4.3 覆盖层系统

| 覆盖层 | 用途 |
|--------|------|
| `overlay-start` | 开始界面，展示规则说明 |
| `overlay-wave` | 波次通关，显示成绩和下一关预告 |
| `overlay-pause` | 暂停，可继续或重新开始 |
| `overlay-end` | 终局，显示总分和再来一局 |

## 5. 游戏流程

```
启动 → 初始化棋盘 → requestAnimationFrame 主循环

每回合:
  玩家操作（交换宝石）
  → 消除判定（匹配或不匹配回弹）
  → 伤害计算 + 英雄充能
  → 特殊宝石展开（火箭/炸弹）
  → 下落填充
  → 连环消除检测
  → 回合结束
  → 敌人计时器累加 → 满 → 敌人攻击
  → 英雄死亡检测 → 团灭 → 失败

波次结束:
  → 敌人 HP=0 → 波次通关弹窗
  → 下一波敌人登场 / 全部击败 → 胜利
```

# Part 2: 技术架构速查

## 6. 架构概览

- **编程范式**: 无 class，纯函数 + 全局状态对象
- **渲染方案**: Canvas（棋盘）+ DOM 面板（敌人/英雄/覆盖层）
- **驱动方式**: `requestAnimationFrame(gameLoop)` 帧循环

## 7. 全局状态

```js
board[8][8]           // 棋盘宝石值 0~4，-1=空
specials[8][8]        // 0=普通, 1=火箭, 2=炸弹
anim                  // 动画状态对象 {phase, elapsed, elimSet, dropCols, ...}
heroStates[]          // 英雄运行时（含 poisoned/poisonedDmg）
// 敌人战斗状态
enemyHP/enemyMaxHP/shield/score/turnCount/waveCount/comboCount
enemyStunned/enemyPoison/enemyPoisonDmg/enemyAtkDown/enemyDefUp
currentEnemyIdx/enemyTimer/frameCount
```

## 8. 动画状态机

```
idle → swapping(180ms) → eliminating(200+180ms) → dropping(300ms) → idle
         ↕ swapBack(180ms, 交换无效时)              ↑ 连环消除时重回 eliminating
```

动画常量: `ANIM_SWAP=180, ANIM_SWAP_BACK=180, ANIM_ELIM_FLASH=200, ANIM_ELIM_SHRINK=180, ANIM_DROP=300`

## 9. 关键函数索引

**棋盘**: `initBoard` / `getMatches` / `createSpecials` / `removeAndDrop` / `swap` / `isAdjacent`

**动画**: `startSwapAnimation` → `startSwapBackAnimation` → `startEliminateAnimation` → `startDropAnimation` → `finishTurn`

**战斗**: `processMatchesAnimated` / `dealDamageToEnemy` / `damageHero` / `enemyAttack` / `useHeroSkill`

**渲染**: `drawBoard`（Canvas 宝石+粒子+浮字）/ `renderHeroCards` / `updateHeroUI` / `updateUI`

**输入**: `cellFromEvent`（坐标转换）/ `handleDragEnd`（拖拽结算，含斜向容错）

**流程**: `startGame` / `restartGame` / `initGameState` / `togglePause`

## 10. gameLoop 核心逻辑

```js
gameLoop(ts):
  if paused → skip
  anim.elapsed += dt → switch phase (swapping/eliminating/dropping)
  粒子和浮动文字逐帧更新
  frameCount++ → 中毒每30帧扣血 / 状态持续帧--
  idle阶段 → enemyTimer++ / ≥200帧 → enemyAttack()
  存活检测 → 团灭 → onPlayerDefeated
  drawBoard()
```
