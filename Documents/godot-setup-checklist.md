## 一、安装 Godot

1. 访问 [godotengine.org/download](https://godotengine.org/download)
2. 下载 **Godot 4.x Stable** 版本（标准版即可，不需要 .NET 版）

## 四、推荐学习路线（按顺序）

### 第 1 步：官方入门教程「躲避怪物」
- 在 Godot 编辑器内 **项目 → 模板 → Dodge the Creeps**
- 或访问官方文档：https://docs.godotengine.org/en/stable/getting_started/first_2d_game/
- 跟着做一遍，你会接触到：场景、节点、脚本、碰撞检测、信号、UI

### 第 2 步：理解核心概念

| 概念 | 一句话解释 |
|------|-----------|
| **节点（Node）** | 游戏里一切物体的基本单位（角色、碰撞体、声音、UI） |
| **场景（Scene）** | 一组节点的集合，保存为 `.tscn` 文件。主角、敌人、关卡各是一个场景 |
| **信号（Signal）** | 节点之间发消息的机制。比如「玩家碰到了道具」→ 发射信号 → 道具消失并生效 |
| **脚本（Script）** | 写 GDScript 代码的地方，挂载在节点上控制行为 |

### 第 3 步：学习 GDScript 基础
- GDScript 语法接近 Python，非常简洁
- 重点掌握：
  - `_process(delta)` — 每帧调用的函数（实时操作）
  - `_input(event)` — 处理键盘/鼠标输入
  - `_ready()` — 节点初始化时调用一次
  - 变量、函数、if/for 循环

### 第 4 步：学习 TileMap（瓦片地图）
- TileMap 用于快速搭建关卡地图（格子拼接地板、墙壁等）
- 以撒类型的地牢地图，用 TileMap 非常合适
- 需要先准备瓦片集（TileSet），也可以先用纯色方块占位

### 第 5 步：了解关键节点类型
- **CharacterBody2D** — 用于玩家/敌人（物理移动 + 碰撞）
- **Area2D** — 检测区域（道具拾取、攻击范围、门触发）
- **Sprite2D** — 显示图片
- **CollisionShape2D** — 碰撞形状（矩形/圆形）
- **Timer** — 延时触发（回合间隔、buff 倒计时）

## 五、核心机制：半回合制的技术实现思路

### 本质
你的「子弹时间半回合制」本质是传统 Roguelike 的 **"你动我才动"** 机制：

- 玩家每回合 = 移动 ~2 步 + 射击 ~1 次（消耗行动点数）
- 行动点数用完 → 敌人立刻依次行动
- 敌人行动完 → 回到玩家回合

### 核心循环伪代码

```gdscript
var player_turn_finished = false

func _process():
    if not player_turn_finished:
        return  # 等待玩家消耗行动点数

    # 玩家回合结束，敌人行动
    for enemy in enemies:
        enemy.take_turn()

    # 所有敌人行动完毕，重置玩家回合
    player_turn_finished = false
    player.move_points = 2
    player.shoot_points = 1
```

### 需要实现的关键模块
- **行动点数系统**（move_points / shoot_points）
- **回合管理器**（控制玩家回合 ↔ 敌人回合切换）
- **敌人 AI**（每个敌人 take_turn 时的行为逻辑）
- **输入拦截**（敌人行动期间禁止玩家操作）

## 七、正式开工前的构思清单

> 建议先用文字把以下内容确定下来，能大大减少后期返工：

1. **主角设定**：外观、起始属性、可成长方向
2. **核心操作**：WASD 移动 / 方向键射击 / 道具快捷键？
3. **房间与关卡**：房间大小？多少房间？门的位置？小地图？
4. **道具系统**：首批想做多少种道具？效果怎么分类（被动/主动/一次性）？
5. **怪物列表**：首批想做多少种怪物？各自的行动模式？
6. **胜利/失败条件**：击败 Boss？清空房间？死亡惩罚？
7. **视觉风格**：像素风？手绘风？先用 Placeholder 方块占位？
