## 时间线

### 1. 道具系统实现

**目标**：基于半回合制以撒玩法，验证 Roguelike 成长体验。快速抓取 wiki 道具数据，精选被动道具做可玩 demo。

**实现**：
- 创建 `isaac-turnbased-demo2.html`（从 demo.html 复制），不影响原文件
- 新增 `ITEMS_DB`，25 个被动道具分 3 个品质等级：
  - 普通（15个）：伤心洋葱(射速+1)、五芒星(攻击+1)、早餐(HP+1)等
  - 稀有（7个）：光环(全属性)、魔法蘑菇(全属性大幅)等
  - 传说（3个）：圣心(攻击+2.5 回血)、蟋蟀头(伤害×1.5)、死神的镰刀(穿透)
- 道具掉落规则：宝箱房必定稀有、Boss房清怪后掉落（20%传说/50%稀有）
- 拾取交互：F键 → 属性叠加 + 道具栏 UI
- 特殊效果：穿透子弹（`player.hasPiercing`）、伤害倍率
- 品质视觉区分：传说道具金色边框+呼吸光效、稀有蓝色边框

### 2. AP 动态绑定属性

**问题**：M-AP 和 A-AP 固定不变，道具提升属性后 AP 没变化。

**修复**：`resetTurnAP()` 中 `maxMAP = Math.floor(moveSpeed)`、`maxAAP = Math.floor(fireRate)`，拾取道具后重新计算。

### 3. 已进入房间不再刷怪

**实现**：新增 `visitedRooms`（Set），`finishTransition` 检测重复进入，跳过 `spawnRoomMonsters()`。`resetGameToFloor1` 重置访问记录。

### 5. 编辑器文件管理重构

**问题**：`isaac-map-viewer.html` 依赖 `node Configs/server.js` 读写文件，但本机未装 Node.js。尝试 PowerShell server.ps1/Python server.py 均不成功。

**最终方案**：彻底移除服务器依赖，改用 File System Access API：

- 楼层数据：新增 **📂 选择 floor-data.json** + **📄 生成json** 按钮

- `showOpenFilePicker` + `<input type="file">` 双重兜底
- `BroadcastChannel('isaac-floors')` 保留：编辑器 → 项目端实时通知

**清理**：删除 `Configs/server.js`、`Configs/test-save.html`、`Configs/server.py`、`Configs/server.ps1`

1. **demo2 独立文件**：不在原 demo.html 上改动，新建文件实验道具系统
