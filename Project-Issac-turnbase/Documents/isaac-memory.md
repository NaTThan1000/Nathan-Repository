## 1. 核心战斗系统

### 1.1 AP 行动点数系统

**2026-07-13 初始**：预操作队列系统
- 玩家先规划本回合所有操作（移动队列 + 射击方向），填入队列后 Space 统一执行
- 执行阶段按队列顺序逐步执行，无暂停机会

**2026-07-15 重构**：即时操作模型（当前方案）
- **[决策]** 彻底移除预操作队列，改为即时操作 + 快照系统
- **[原因]** 预操作队列体验差，缺乏容错能力，玩家无法在执行中调整决策
- **[方案]**
  - WASD 在 BFS 可移动范围（浅蓝呼吸闪烁）内即时移动本体
  - ↑↓←→ 即时射击（按即发射）
  - Esc 全重置：通过回合快照系统 `saveTurnSnapshot()` / `restoreTurnSnapshot()`，恢复玩家/怪物/HP 全部到回合开始时
  - 首次射击触发 checkpoint（`hasShot` + `checkpointPos`），刷新可移动范围从当前位置重新计算
  - 回合起始位置保留 40% 透明度幽灵作视觉参考
- **[属性调整]** 移速=3 → M-AP=3，射速=3 → A-AP=3，系数 1，下限 1

**2026-07-20 AP 动态绑定**：`maxMAP = Math.floor(moveSpeed)`、`maxAAP = Math.floor(fireRate)`
- **[原因]** 道具系统改变属性后 AP 需同步变化
- **[规则]** 向下取整保证整数步数，道具加成可能为小数

### 1.2 无敌系统 ← 反复变更

**2026-07-16 初始**（双轨制）：
- 战斗模式：受伤后无敌 3 回合（`invincibleTurns`）
- 探索模式：受伤后无敌 `invincibleSteps = 移速×2` 步（每移动一格递减）

**2026-07-20 统一**（当前方案）：
- **[决策]** 移除 `invincibleTurns`，战斗/探索统一使用 `invincibleSteps = 移速×2` 步数制
- **[原因]** 战斗↔探索切换时无敌持续逻辑不一致
- **[关键规则]** 战斗模式结束回合时，未消耗的 M-AP 也计入无敌步数消耗：`invincibleSteps -= turnState.mAP`（只读 mAP 值，不改变 AP 系统）
- **[示例]** 移速=3，受伤 → 无敌 6 步。战斗中移动了 1 步结束回合，剩余 2 M-AP → 再减 2 步，无敌还剩 3 步

## 2. 怪物系统

### 2.1 配置演变

**2026-07-14 初始**：`MONSTER_CFG` — 单一裂口尸，内联 JS 对象

**2026-07-16 重构**（两阶段）：
- **阶段一**：`MONSTER_DB` 内联 JS，4 种怪物（裂口尸/浮游眼/岩石魔像/Boss裂口之王），6 种 AI 行为类型
- **阶段二**：外置 `Configs/monster-db.json`，新增 `movementTags`（地面/飞行）、`role`（melee/ranged/tank/boss）、`threat`（威胁值）三个字段
- **[原因]** 计划文档建议支持多种怪物类型，外置 JSON 便于配置和编辑器修改

### 2.2 刷怪算法

**2026-07-16**：三层递进算法
- 标签过滤：怪物 `movementTags` ∩ 房间 `allowedMovement`
- 组合规则：至少 1 近战保底 + 角色多样性加权（×3）
- 点数预算：基础 budget + (楼层-1) × 2

**2026-07-20 细化**：
- C 键/按钮：95% 普通池 + 5% 含 Boss 随机
- Boss 房固定生成 `boss_maw_king`，起点房无怪
- `visitedRooms`（Set）：已进入房间不再刷怪，避免反复刷

### 2.3 视觉与渲染

**2026-07-16**：
- 怪物渲染复用 `drawCharacterAt()`，增加 `tint` 参数做色彩叠加（浮游眼蓝紫半透/魔像棕半透/Boss红橙半透）
- 头顶名称标签（Boss 红字突出，普通棕色）
- 红色血条（黑色底槽 + 百分比填充），受击白闪 + 抖动
- 路径预览：红色粗箭头闪烁（仅玩家回合）
- 伤害反馈：黄色飘字迸发 + 血条白色延迟扣除动画 + 血雾粒子
- 死亡：18 颗血粒子爆浆特效

### 2.4 怪物属性

- 每只怪物独立 `moveCycle`（移速周期数组），不再全局共享
- 每只怪物独立 `damage`（碰撞伤害）
- `aiType`：AI 行为路由，`calcAllMonsterPaths()` 中 switch 分发
- 怪物之间不可重叠；撞玩家 → 伤害按怪物类型区分（裂口尸/浮游眼=1，魔像/Boss=2）+ 击退
- 尖刺地格：怪物经过 → 受到 5 点伤害（2026-07-20 从无尖刺判定新增）

## 3. TILE 瓷砖系统

### 3.1 尖刺伤害 ← 反复调整

**2026-07-15 初始**：5 种 TILE（FLOOR/ROCK/POOP/PIT/SPIKE），尖刺伤害未细化

**2026-07-16**：尖刺伤害 0.5（不明确）

**2026-07-20 修正**：尖刺伤害 玩家 2 / 怪物 5
- **[原因]** 之前 0.5 无法实际扣除战斗整数 HP

### 3.2 透视/房门

**2026-07-16**：
- 透视取消：`VP_SCALE_TOP = VP_SCALE_BOT = 1.00`，标准矩形渲染，无透视变形
- 房门重绘：开=深色通道+门框，关=铁栏纹理+横竖铁条，区分上下/左右方向
- 移除预览线/透视描述等已淘汰旧内容

### 3.3 其他 TILE

- ROCK `#`：阻挡通行和子弹
- POOP `P`：阻挡，可受击 3 次破坏
- PIT `_`：深渊，踩上即死或掉落
- LADDER `▼`：Boss 房 B 键放置，踩上 `enterFloor()` 进入下一层

## 4. 房间与楼层系统

### 4.1 多房间框架

**2026-07-14 方案**：创建 `isaac-roommonster-plan.md`，定义楼层生成算法（随机图+BFS全连通）、房间类型（start/normal/treasure/shop/boss）、模板法内部布局

**2026-07-15 实现**：12 种房间模板，`isaac-map-viewer.html` 编辑器

**2026-07-16 强化**：
- 房间类型约束：Boss 房和宝箱房强制度数=1（单门死路），保证至少 3 个叶节点
- 所有房间度数≤4（每方向最多一扇门）
- 布局强制相邻：Manhattan 距离=1，随机重试最多 30 次
- 门一致性校验
- 楼层 grid 固化：加载 `floor-data.json` 时保留已存 grid，修改 `pool.json` 不影响已生成楼层

### 4.2 编辑器文件管理 ← 反复纠结

**2026-07-16 初始**：`node Configs/server.js` 提供文件读写服务

**2026-07-20 多次尝试**：
- 尝试 PowerShell：失败（未装相关模块）
- 尝试 Python `server.py`：失败
- 尝试 `<form>` POST + 隐藏 `<iframe>` 绕过 IDE 代理拦截：不可靠

**最终方案（2026-07-20）**：彻底去服务器
- **[决策]** 使用 File System Access API 直读直写 JSON 文件
- `showOpenFilePicker` + `<input type="file">` 双重兜底
- "生成 json"按钮 → 弹窗显示 JSON 文本 → 用户手动复制粘贴覆盖
- IndexedDB 持久化文件句柄，下次自动记住
- **[清理]** 删除 `Configs/server.js`、`Configs/test-save.html`、`Configs/server.py`、`Configs/server.ps1`
- **[结论]** 不装 Node/Python 时最可靠方案

## 5. 道具系统 [demo2.html 专属]

### 5.1 设计与实现 [2026-07-20]

- **[决策]** 创建 `isaac-turnbased-demo2.html`（v3 道具版），不在原 demo.html 上改动
- **25 种被动道具**，分 3 个品质等级：

| 品质 | 数量 | 示例 |
|------|------|------|
| 普通 | 15 | 伤心洋葱(射速+1)、五芒星(攻击+1)、早餐(HP+1) |
| 稀有 | 7 | 光环(全属性)、魔法蘑菇(全属性大幅) |
| 传说 | 3 | 圣心(攻击+2.5 回血)、蟋蟀头(伤害×1.5)、死神的镰刀(穿透) |

- **掉落规则**：
  - 宝箱房：必定掉落稀有道具
  - Boss 房清怪后：20% 传说 / 50% 稀有 / 30% 普通
- **拾取**：F 键交互 → 属性叠加 + 道具栏 UI（底部图标 + 悬浮提示）
- **品质区分**：传说=金色边框+呼吸光效、稀有=蓝色边框、普通=棕色边框
- **特殊效果**：穿透子弹（`player.hasPiercing` → 丘比特之箭/死神的镰刀）、伤害倍率（蟋蟀头 ×1.5）

### 5.2 原始 demo 文件保护 [2026-07-20]

- 多次误改 `isaac-turnbased-demo.html`，最终通过 `git checkout` 还原
- **[规则]** 后续所有改动仅在 demo2 和 map-viewer 中进行

## 7. 文档与记忆体系

### 7.1 建立三层记忆体系 [2026-07-20]

- **[问题]** 多台电脑开发，CodeBuddy 对话不在不同电脑间同步
- **[方案]**
  - L1 `Documents/global-rules.md`：跨项目通用规范
  - L2 `{project}/Documents/{project}-memory.md`：项目决策记忆（本文档）
  - L3 `{project}/Documents/{project}-context.md`：策划文档 + 技术速查
- **[大小决策]** 不分月，单文件至少撑 2-3 年
- **[更新时机]** 合并 main 流程中统一更新；如有特殊需要可在切换电脑前临时追加
- **[使用方式]** 不同电脑的新会话启动时，用户主动告诉 AI 读取三文件

### 7.2 CodeBuddy Memories 互补 [2026-07-20]
- Memories 存储精简版规则（自动加载作快速索引）
- global-rules.md 存储详细版（含时间戳和上下文）
- 两者可互相校验保证一致
